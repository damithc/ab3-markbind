    Documentation Guide

    We use MarkBind to manage documentation. MarkBind is a tool for generating static websites from markdown-like text, particularly suitable for text-heavy websites such as software project documentation.

    The docs/ folder contains the source files for the documentation website.

    Installing MarkBind

    For most non-trivial changes, you would want to edit the source files locally, and preview the generated site first (to ensure the edits produces the intended outcome), before publishing the changes. For that, you need to install MarkBind in your computer. Given below are the steps.

    1. Ensure you have Java 8 or higher installed
    2. Install Node.js 16 or higher
    3. Install Graphviz (for generating diagrams)
      • ​
        Windows: you can use Chocolatey to install Graphviz: choco install graphviz
      • ​
        Mac: you can use Homebrew to install Graphviz: brew install graphviz
      • ​
        For Linux, you can use your package manager to install Graphviz: sudo apt install graphviz
    4. Open a terminal and navigate to the [project root]/docs folder
    5. To install MarkBind (and other dependencies), run the command npm ci

    To update MarkBind to a newer version (at a later time), run the following command from the docs folder:

    npm install markbind-cli@latest --save-dev

    Updating documents

    MarkBind is a superset of Markdown. Refer the MarkBind UG: Authoring Contents for more details on the syntax. Given below is the recommended workflow for any non-trivial changes to this docs site.

    1. Open a terminal and navigate to the [project root]/docs folder.
    2. Run the npm run serve command. That will open the generated website in your default browser in a live preview mode.
    3. In the browser, navigate to the page you want to preview.
    4. Edit the source files (usually, .md files). When you save the file, the live preview will update to reflect the new contents (after a few seconds).

    While live preview can pick up most changes, it may not be able to pick up certain changes (e.g., changes to files in the _markbind folder or changes to nunjucks macros). Furthermore, some syntax errors in your code can cause the live preview to crash. In those cases, just stop the server (e.g., Ctrl+C on Windows) and start it again.

    If you are using Intellij for editing documentation files, you can consider enabling 'soft wrapping' for *.md files, as explained in se-edu/guides Intellij IDEA: Useful settings

    Adopting to a new fork

    When adapting the documentation to a new fork that evolves/morphs AB-3 into a different project/product, in addition to updating content files, you need to update the following config files:

    [project root] ├── .github │ └── workflows │ └── docs.yml └── docs ├── _markbind │ └── layouts │ └── default.md └── site.json

    The markbind/tweaking the page structure and markbind/site.json pages have information on how to update site-wide elements such as the top navigation bar.

    Deploying updates

    This project is configured to deploy the docs site automatically, using the GitHub Actions CI pipeline (the configuration is specified in the .github/workflows/docs.yml file). All you need to do is to push the updated source files to the repo and wait for the GitHub Actions to run. Once ready, the updated site will be available at https://github.io/ORG/REPO e.g., https://se-education.org/addressbook-level3

    More info about deployment options can be found in this part of the MarkBind UG.
